## Overview

JobApply runs a four-phase pipeline fully automated from discovery to application:

| Phase | What It Does |
|-------|-------------|
| **1. Discovery** | Scrapes intern-list.com (via jobright.ai embed) and LinkedIn Easy Apply listings using Playwright. Stores all jobs in SQLite. |
| **2. Filter / Score** | Keyword-based scoring ranks jobs by AI/ML relevance and recency. Sponsorship-required and clearance-required postings are excluded automatically. |
| **3. Tailor** | Fetches full job descriptions, calls Gemini 2.5 Flash to produce a tailored resume JSON and cover letter, and renders both to PDF via ReportLab. |
| **4. Apply** | Playwright auto-fills Greenhouse, Lever, and LinkedIn Easy Apply forms. Always pauses for human confirmation before submitting. |

---

## Features

- **Multi-source scraping** — intern-list.com and LinkedIn in a single run
- **Smart scoring** — weighted keyword scoring with a recency bonus
- **Auto-exclusion** — filters out "no sponsorship", "clearance required", and similar blockers
- **LLM tailoring** — per-job resume and cover letter generated by Gemini 2.5 Flash (with Claude fallback)
- **PDF generation** — professional PDFs rendered with ReportLab, committed back to the repo by CI
- **ATS auto-fill** — supports Greenhouse, Lever, and LinkedIn Easy Apply
- **Human-in-the-loop** — apply step always pauses for your confirmation before submitting
- **No fabrication** — LLM is explicitly instructed never to invent skills or experience
- **Deduplication** — SQLite UNIQUE constraint on job URL prevents duplicate entries
- **Daily CI pipeline** — GitHub Actions discovers and tailors up to 50 jobs every morning
- **Streamlit dashboard** — browse, review, and trigger tailoring from a hosted web UI

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                        JobApply Pipeline                        │
└─────────────────────────────────────────────────────────────────┘

  ┌─────────────────┐    ┌─────────────────┐
  │  intern-list.com│    │    LinkedIn      │
  │  (jobright.ai)  │    │  (Easy Apply)   │
  └────────┬────────┘    └────────┬────────┘
           │   Playwright          │   Playwright
           └──────────┬────────────┘
                      ▼
           ┌─────────────────────┐
           │   SQLite Tracker    │  ← status: new
           │  applications.db    │
           └──────────┬──────────┘
                      │
          ┌───────────▼────────────┐
          │   Phase 2: Filter /    │
          │        Score           │
          │  • AI/ML keywords      │
          │  • Recency bonus       │
          │  • Exclude blockers    │
          └───────────┬────────────┘
                      │ top-N jobs
          ┌───────────▼────────────┐
          │   Phase 3: Tailor      │
          │  jd_fetcher (BS4)      │
          │  → Gemini 2.5 Flash    │
          │  → Resume JSON         │
          │  → Cover Letter text   │
          │  → ReportLab PDF       │
          └───────────┬────────────┘
                      │ status: queued
          ┌───────────▼────────────┐
          │   Phase 4: Apply       │
          │  Playwright ATS fill   │
          │  [PAUSE — confirm?]    │
          │  Greenhouse / Lever /  │
          │  LinkedIn Easy Apply   │
          └───────────┬────────────┘
                      │ status: applied / rejected
          ┌───────────▼────────────┐
          │  Streamlit Dashboard   │
          │  Review · Stats · Tag  │
          └────────────────────────┘
```

### Scoring formula

```
score = strong_keyword_match (up to 0.45)
      + good_keyword_match   (up to 0.30)
      + recency_bonus        (up to 0.15)
```

Jobs are sorted descending by score; only the top-N proceed to tailoring.

## Setup

### Prerequisites

- Python 3.12+
- A Google AI Studio API key — [aistudio.google.com](https://aistudio.google.com)
- (Optional) LinkedIn account credentials for LinkedIn scraping

### 1. Clone the repository

```bash
git clone https://github.com/ikathuria/JobApply.git
cd JobApply
```

### 2. Install dependencies

Playwright is installed separately to maintain Streamlit Cloud compatibility.

```bash
pip install -r requirements.txt playwright
playwright install chromium
```

### 3. Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in:

```env
GOOGLE_API_KEY=your_key_from_aistudio
LINKEDIN_EMAIL=your@email.com
LINKEDIN_PASSWORD=yourpassword
```

### 4. Configure your profile

Edit `config/profile.json` with your resume data — name, education, work experience, skills, and work authorization status. This file is the single source of truth the LLM uses when tailoring your resume.

### 5. (Optional) Adjust scoring and sources

Edit `config/settings.yaml` to change scoring weights, keyword lists, job sources, or switch the LLM provider.

## Usage

All commands run from the project root.

```bash
# Discover jobs from all configured sources
python main.py

# Discover from a specific source only
python main.py --source intern_list
python main.py --source linkedin        # prompts for manual 2FA if needed

# Tailor the top 50 new/unprocessed jobs
python main.py --tailor --limit 50

# Auto-apply to up to 10 queued (tailored) jobs
python main.py --apply --limit 10

# Show application tracker statistics
python main.py --stats

# Launch the Streamlit review dashboard (local)
streamlit run dashboard/app.py
```

### Application status flow

```
new  →  queued  →  applied  →  interview
                          ↘  offer
                          ↘  rejected
```

## GitHub Actions

A daily workflow runs automatically at **9:00 AM UTC** every day.

**What it does:**
1. Scrapes intern-list.com for new listings
2. Scores and filters jobs
3. Tailors the top 50 jobs (Gemini 2.5 Flash)
4. Renders PDFs to `output/resumes/`
5. Commits the updated `tracker/applications.db` and new PDFs back to the repository

**Required repository secret:**

| Secret | Description |
|--------|-------------|
| `GOOGLE_API_KEY` | Your Google AI Studio API key |

Set this under **Settings → Secrets and variables → Actions → New repository secret**.

LinkedIn scraping is intentionally excluded from the GHA workflow because it requires interactive 2FA. Run `python main.py --source linkedin` locally when you want to pull LinkedIn listings.

## Streamlit Dashboard

The dashboard is deployable to [Streamlit Community Cloud](https://share.streamlit.io) and reads from the `tracker/applications.db` file committed to the repository by GitHub Actions.

**Features:**
- Browse all discovered jobs with scores and statuses
- View generated resume and cover letter for any tailored job
- Trigger tailoring for individual jobs (cloud-compatible)
- Filter by status, score range, or keyword
- View summary statistics

**Cloud-only limitations:**

| Feature | Local | Streamlit Cloud |
|---------|-------|-----------------|
| Discovery (scraping) | Yes | No (Playwright not available) |
| Tailor | Yes | Yes |
| Apply | Yes | No (Playwright not available) |
| Review / Stats | Yes | Yes |

**Deploy to Streamlit Cloud:**
1. Push the repo to GitHub
2. Connect it at share.streamlit.io
3. Add `GOOGLE_API_KEY` to **App settings → Secrets**

## Configuration

### `config/settings.yaml`

```yaml
sources:
  intern_list: true
  linkedin: true

scoring:
  strong_keywords:     # up to 0.45 score contribution
    - LLM
    - RAG
    - machine learning
    - deep learning
    # ...
  good_keywords:       # up to 0.30 score contribution
    - Python
    - NLP
    - transformer
    # ...
  recency_bonus: 0.15  # added for jobs posted within N days

exclusions:
  - "no sponsorship"
  - "sponsorship not available"
  - "clearance required"
  - "security clearance"

llm:
  provider: gemini              # or: anthropic
  model: gemini-2.5-flash
  thinking_budget: 0            # 0 disables thinking to avoid token truncation
  rate_limit_sleep: 2           # seconds between API calls
```

### `config/profile.json`

Contains all candidate data used by the LLM to tailor applications:

```json
{
  "name": "Ishani Kathuria",
  "email": "...",
  "education": [...],
  "experience": [...],
  "skills": [...],
  "work_authorization": "OPT / F-1 student (no sponsorship required for internships)"
}
```

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Human-in-the-loop apply** | The apply step always pauses for confirmation before submitting any form. No accidental submissions. |
| **No LLM fabrication** | The system prompt explicitly prohibits inventing skills, experience, or credentials. Tailoring only reframes real content. |
| **SQLite over a remote DB** | Keeps the project self-contained and portable; the DB is small enough to commit to git. |
| **Playwright separate from requirements.txt** | Playwright has OS-level browser dependencies that break Streamlit Cloud installs if included in the main requirements file. |
| **thinking_budget=0** | Disables chain-of-thought on structured JSON tasks, preventing token budget overruns that would truncate output. |
| **2s sleep between Gemini calls** | Stays comfortably within the paid tier limit of 1,000 RPM / 10,000 RPD. |

## Contributing

1. Fork the repository
2. Replace `config/profile.json` with your own resume data
3. Adjust keywords in `config/settings.yaml` to match your target roles

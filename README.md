An end-to-end automated AI/ML internship application system. JobApply discovers job listings, scores and filters them, enriches job details via Jobright, generates tailored resumes and cover letters with an LLM, and walks you through an interactive apply session with a headed browser — pausing for your confirmation before any form is submitted.

Built for Ishani Kathuria (MS AI @ Purdue, ex-AWS SDE) targeting internship roles in the USA, with AI/ML relevance handled by scoring.

## Overview

JobApply runs a four-phase pipeline from discovery to application:

| Phase | What It Does |
|-------|-------------|
| **1. Discovery** | Scrapes intern-list.com, LinkedIn Easy Apply, and Handshake using Playwright. Stores all jobs in Turso (cloud) or SQLite (local). |
| **2. Filter / Score** | Keyword-based scoring ranks jobs by AI/ML relevance, role type, recency, and sponsorship-friendliness. Blockers (visa-required, clearance-required) are auto-excluded. |
| **3. Enrich + Tailor** | Fetches full job descriptions; Jobright detail pages are enriched to resolve real employer URLs and salary info. Gemini 2.5 Flash then produces a tailored resume JSON and cover letter, rendered to PDF via ReportLab. |
| **4. Apply** | Playwright fills Greenhouse, Lever, and LinkedIn Easy Apply forms in a headed browser. You review the filled form and press Enter to submit — or skip. |

---

## Features

- **Multi-source scraping** — intern-list.com (via Jobright embed), LinkedIn Easy Apply, and Handshake in a single run
- **Smart scoring** — weighted keyword + role relevance + recency + sponsorship-friendliness
- **Auto-exclusion** — filters out "no sponsorship", "clearance required", and similar blockers before any tailoring work happens
- **Jobright enrichment** — resolves Jobright detail pages to real employer ATS URLs, salary ranges, and updated descriptions; marks closed postings as skipped automatically
- **LLM tailoring** — per-job resume and cover letter generated by Gemini 2.5 Flash (with Claude fallback); prompt explicitly prohibits fabricating skills or experience
- **PDF generation** — professional resume and cover letter PDFs rendered with ReportLab, committed back to the repo by CI
- **ATS auto-fill** — supports Greenhouse, Lever, and LinkedIn Easy Apply; detects 10+ other platforms (Workday, Ashby, SmartRecruiters, iCIMS, Taleo, Jobvite, Rippling) for manual fallback
- **Interactive apply** — always opens a headed browser; you solve any CAPTCHA, review the filled form, and confirm before any submission
- **Cloud persistence** — Turso (libSQL) keeps the job database in sync across local machine, CI, and the deployed web app; falls back to SQLite when Turso credentials are absent
- **React dashboard** — warm paper design with a Command Center overview, pipeline tabs, swipe-style review decks, tab-scoped search + filter bar, and analytics
- **Daily CI pipeline** — GitHub Actions discovers and tailors up to 50 jobs every morning, commits new PDFs
- **Deduplication** — UNIQUE constraint on job URL prevents duplicate entries across scraping runs
- **CSV import** — bulk-import jobs from any spreadsheet export via the dashboard

                        │  → Resume JSON      │
                        │  → Cover Letter     │
                        │  → ReportLab PDF    │
                        └──────────┬──────────┘

                        ┌──────────▼──────────┐

                        │  Playwright (headed)│

                        │  LinkedIn Easy Apply│

### Application status flow

```
new → queued → approved → applied → oa → interview → offer
  ↘ skipped              ↘ rejected
```

### Scoring formula

```
score = keyword_match        (weight 0.40)
      + role_relevance       (weight 0.30)
      + recency_bonus        (weight 0.20)
      + sponsorship_friendly (weight 0.10)
```

Jobs are sorted descending by score; only the top-N proceed to tailoring each run.

## Setup

### Prerequisites

- Python 3.12+
- Node.js 18+ and npm
- A Google AI Studio API key — [aistudio.google.com](https://aistudio.google.com)
- (Optional) Turso account for cloud database sync — [turso.tech](https://turso.tech)
- (Optional) LinkedIn / Handshake credentials for those scrapers

### 1. Clone the repository

```bash
git clone https://github.com/ikathuria/JobApply.git
cd JobApply
```

### 2. Install Python dependencies

```bash
pip install -r requirements.txt
playwright install chromium
```

Playwright is pinned in `requirements.txt`; the second command installs the Chromium browser binary.

### 3. Install frontend dependencies

```bash
cd web && npm install && cd ..
```

### 4. Configure environment variables

Create a `.env` file in the repo root:

```env
# Required
GOOGLE_API_KEY=your_key_from_aistudio

# Required for cloud DB sync (omit to use local SQLite)
TURSO_DATABASE_URL=libsql://your-db.turso.io
TURSO_AUTH_TOKEN=your_turso_token

# Optional — needed for LinkedIn scraping / apply
LINKEDIN_EMAIL=your@email.com
LINKEDIN_PASSWORD=yourpassword

# Optional — needed for Handshake scraping
HANDSHAKE_EMAIL=your@email.com
HANDSHAKE_PASSWORD=yourpassword

# Optional — Claude fallback for LLM tailoring
ANTHROPIC_API_KEY=your_anthropic_key
```

When `TURSO_DATABASE_URL` is absent the pipeline falls back to a local `tracker/applications.db` SQLite file automatically.

### 5. Configure your profile

Edit `config/profile.json` with your resume data — name, contact info, education, work experience, skills, and work authorization status. This is the single source of truth the LLM uses when tailoring your resume and filling application forms.

### 6. Adjust sources and scoring

Edit `config/settings.yaml` to enable/disable scrapers, change scoring weights, keyword lists, or switch the LLM provider.

## Usage

### Discovery + tailoring (CLI)

```bash
# Discover jobs from all configured sources
python main.py

# Single source
python main.py --source intern_list
python main.py --source linkedin      # headed browser; handles 2FA interactively
python main.py --source handshake

# Tailor the top N new jobs (fetch JD + generate resume + cover letter PDFs)
python main.py --tailor --limit 50

# Enrich priority jobs — resolves Jobright URLs to real employer ATS URLs
python main.py --resolve-priority-links

# Show application tracker statistics
python main.py --stats
```

### Interactive apply

After reviewing and approving jobs in the dashboard:

```bash
# Opens a headed browser; fill any missing fields, confirm each submit
python main.py --apply --limit 10

# Dry run — fills forms and takes screenshots, never submits
python main.py --apply --dry-run --limit 10
```

The apply loop:
1. Navigates to the employer ATS URL
2. Detects the ATS platform (Greenhouse / Lever / LinkedIn / or prompts you to fill manually)
3. Auto-fills contact info, uploads resume PDF, pastes cover letter
4. Pauses — you review, solve any CAPTCHA, then press **Enter** to submit or **s** to skip

### Dashboard (local dev)

```bash
# Terminal 1 — FastAPI backend
uvicorn api.main:app --reload --port 8000

# Terminal 2 — React dev server
cd web && npm run dev
```

Open [http://localhost:5173](http://localhost:5173). The Vite dev server proxies `/api/*` to `:8000`.

### Production build

```bash
cd web && npm run build && cd ..
uvicorn api.main:app --host 0.0.0.0 --port 8000
```

FastAPI serves the built `web/dist/` bundle at `/` and the API at `/api/*`.

## GitHub Actions

### `daily_tailor.yml` — runs at 9:00 AM UTC daily

1. Discovers new listings from intern-list.com, LinkedIn, and Handshake (each source runs independently with `continue-on-error` so one failure never blocks the others)
2. Scores and filters all new jobs
3. Tailors the top 50 jobs (Gemini 2.5 Flash): fetches JDs, generates resume JSON + cover letter, renders PDFs
4. Commits new PDFs from `output/resumes/` back to the repository

The database itself lives in Turso — only the generated PDFs are committed to git.

**Required repository secrets:**

| Secret | Description |
|--------|-------------|
| `GOOGLE_API_KEY` | Google AI Studio key for Gemini |
| `TURSO_DATABASE_URL` | Turso database URL (`libsql://...`) |
| `TURSO_AUTH_TOKEN` | Turso auth token |
| `LINKEDIN_EMAIL` | LinkedIn account (optional — skip if LinkedIn disabled) |
| `LINKEDIN_PASSWORD` | LinkedIn password (optional) |
| `HANDSHAKE_EMAIL` | Handshake account (optional) |
| `HANDSHAKE_PASSWORD` | Handshake password (optional) |

Set these under **Settings → Secrets and variables → Actions → New repository secret**.

> **Note:** There is no automated apply workflow. The apply step is intentionally interactive-only — run `python main.py --apply` locally when you are ready to submit.

## Configuration

### `config/settings.yaml`

```yaml
sources:
  intern_list:
    enabled: true
  linkedin:
    enabled: true
    search_query: "intern"
    easy_apply_only: true
    max_jobs: 60
  handshake:
    enabled: true
    queries: ["intern", "software intern", "data intern", "AI intern", "machine learning intern"]
    max_jobs: 80

scoring:
  weights:
    keyword_match: 0.4
    role_relevance: 0.3
    recency: 0.2
    sponsorship_friendly: 0.1
  min_score: 0.0     # keep all listings; score used for sorting only

llm:
  provider: "gemini"
  gemini_model: "gemini-2.5-flash"
  anthropic_model: "claude-sonnet-4-6"
```

## Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Interactive-only apply** | Auto-submitting through a headed browser is unreliable — ATS platforms use CAPTCHAs and bot detection that require human intervention. The apply step fills what it can and hands control to you. |
| **No LLM fabrication** | The system prompt explicitly prohibits inventing skills, experience, or credentials. Tailoring only reframes and emphasizes real content. |
| **Turso for cloud, SQLite for local** | Turso (libSQL) provides a persistent cloud replica accessible from CI, the deployed web app, and the local machine simultaneously. Local runs fall back to SQLite automatically when Turso credentials are absent, keeping the project usable offline without any configuration. |
| **React + FastAPI over Streamlit** | Streamlit's execution model made fine-grained UI state (drawers, modals, inline edits, swipe decks) awkward. FastAPI serves as both the API layer and the static file host for the built React bundle, keeping deployment a single process. |
| **`web/dist` committed to git** | Render's free tier has limited build time; committing the pre-built bundle means zero frontend build time on deploy and faster cold starts. |
| **Playwright in `requirements.txt`** | Playwright is pinned alongside all other Python deps so the version is consistent across local installs and CI. The browser binary is installed separately with `playwright install chromium` because it requires OS-level dependencies. |
| **`thinking_budget=0` for Gemini** | Disables chain-of-thought on structured JSON tasks, preventing token budget overruns that would truncate the resume JSON output mid-object. |
| **`continue-on-error` per scraper in CI** | Each discovery source runs as an independent step. A LinkedIn block or Handshake timeout never prevents the other sources from completing or the tailor step from running. |

## Contributing

This is a personal automation project. To adapt it for your own job search:

1. Fork the repository
2. Replace `config/profile.json` with your own resume data
3. Adjust keywords and sources in `config/settings.yaml`
4. Add `GOOGLE_API_KEY` (and optionally `TURSO_DATABASE_URL` + `TURSO_AUTH_TOKEN`) to `.env` locally and to GitHub / Render secrets for CI
5. Run `python main.py` to start your first discovery pass

Bug reports and pull requests are welcome. Please open an issue first for significant changes.
